   Программа Project Expert 7 предназначена для работы на персональных компьютерах 
IBM PC и полностью совместимых с ними компьютерах, с процессором не ниже i486DX-
66 и ОЗУ не менее 32 Мбайт.
   Project Expert 7 работает в операционных системах семейства Windows (начиная с 
Windows 95). В качестве дополнительного программного обеспечения желательно 
наличие на компьютере установленных программ MS Word и MS Excel.
   Для установки программы требуется свободное дисковое пространство не менее 30 
Мбайт (не считая объема, необходимого для хранения файлов с данными).
   Вместе с Project Expert 7 поставляется физический ключ защиты Hardlock, наличие 
которого в LPT или USB портах (порт зависит от модификации ключа) позволяет 
запускаться и работать программе. 


   Порядок установки программы:
   
1.	В порт компьютера устанавливается физический ключ защиты Hardlock.
При этом необходимо выполнять следующие правила:
	а) Подключение или отключение LPT ключа производится только при 
выключенном компьютере.
	б) Не рекомендуется прилагать излишних усилий при установке ключа в порт.
	в) Не допускается загрязнение и механические повреждения разъемов 
электронного ключа.

2.	Устанавливается приложение HL-server. (Если планируется использовать программу 
локально на компьютере, то выполнять этот пункт не следует). 
Если предполагается использовать Project Expert 7 в локальной сети, то необходимо с 
компакт диска с дистрибутивом Project Expert установить на компьютер с ключом защиты 
менеджер лицензий HL-server, запустив установочный файл setup.exe, который 
располагается по пути «CD диск»:\Setup\HL-server. В зависимости от операционной 
системы и от параметров установки HL-server может быть установлен как сервис или как 
приложение. Установленный менеджер лицензий всегда должен быть запущен на 
компьютере с ключом, в противном случае работа Project Expert 7 в локальной сети будет 
невозможна.

3.	Устанавливается приложение Project Expert 7.
Установку самой программы можно осуществить, воспользовавшись или автозапуском 
компакт диска с дистрибутивом, или непосредственным запуском исполняемого файла. В 
первом случае вставляется диск в CD-ROM, после чего автоматически запускается сплэш-
меню, в котором пользователем выбирается верхний пункт Project Expert и начинается 
установка. Во втором случае следует при помощи проводника Windows зайти на компакт 
диск, где выбирается корневая папка Setup, затем содержащаяся в ней папка PE, в которой 
запускается файл установки setup.exe. При установке программы пользователю 
предлагается ознакомиться с лицензионным соглашением, выбрать место расположения 
программы на диске, а также, при необходимости, настроить связь с менеджером 
лицензий.

Внимание! Если предполагается использовать Project Expert 7 в локальной сети, то при 
установке программы необходимо настроить связь с менеджером лицензий HL-Server. Это 
можно осуществить на последнем этапе установки программы при выдаче 
соответствующего запроса или при запуске из папки Project Expert 7 в меню программ 
Windows пункта «Настройка сетевого подключения». При этом в окне «Настройка 
сетевого соединения» указывается IP-адрес или название компьютера с ключом и 
менеджером лицензий. После настроек требуется перезапустить компьютер. Также при 
использовании Project Expert 7 в локальной сети необходимо установить на компьютере с 
ключом защиты приложение HL-server (см. пункт 2).

Внимание! Если программа устанавливается не под Windows 95 или Windows 98, то 
установку необходимо осуществлять с правами локального администратора Windows, при 
этом имя этого администратора должно состоять из английский букв. Если отвечающего 
этим условиям пользователя нет, то его можно временно создать и после установки 
программы удалить.
Чтобы добавить пользователя в Windows 2000/XP, нужно щелкнуть правой кнопкой мыши 
на иконке «Мой компьютер» («My computer»), выбрать во всплывающем меню пункт 
«Управление» («Manage»), в появившемся окне «Управление компьютером» («Computer 
Management») выбрать раздел «Локальные пользователи и группы» («Local Users and 
Groups») и в нем подраздел «Пользователи» («Users»). После выбора последнего 
подраздела нужно в правой части окна щелкнуть мышью на белом поле и во 
всплывающем меню выбрать пункт «Новый пользователь» («New User»). В появившемся 
окне задается имя нового пользователя (оно должно состоять из английских букв – 
например admin) и его пароль, после чего создается пользователь в системе посредством 
нажатия на кнопку «Создать» («Create») и закрывается окно. В результате выполнения 
перечисленных действий в правой части окна «Управление компьютером» («Computer 
Management») появляется только что созданный пользователь. После этого нужно 
щелкнуть правой кнопкой мыши на этом пользователе, выбрать в появившемся меню 
пункт «Свойства» («Properties») и в новом окне на закладке «Членство в группах» 
(«Member Of») добавить группу «Администраторы» («Administrators») посредством 
нажатия на кнопку «Добавить» («Add»).

      Осуществление всех перечисленных выше действий должно позволить запускать 
программу Project Expert 7 из меню программ Windows. Но в случае, когда работа с 
программой осуществляется в локальной сети, в которой используется несколько 
однородных ключей Hardlock для разных программ (например, Project Expert 7 и Audit 
Expert 3), процесс запуска этих программ усложняется.
 
      Для одновременной работы нескольких программ с однородными ключами 
необходимо настроить их таким образом, чтобы запуск происходил со своими 
переменными окружения для каждой. Для этого используется утилита run.exe, 
поставляющаяся вместе с дистрибутивом Projeсt Expert 7 и Audit Expert 3 (она находится в 
каталоге bin, который располагается в каталоге Project Expert 7). Для настройки программ 
необходимо при помощи этой утилиты выполнить следующую последовательность 
действий:
1) Запустить утилиту «run.exe» без параметров. При этом будет создан файл «run.ini», 
содержащий в себе в разделе HL текущие значения переменных окружения.
2) Отредактировать данный файл таким образом, чтобы значение переменных окружения 
соответствовало необходимым для запуска данного программного продукта.
3) Производить дальнейший запуск программных продуктов с помощью утилиты run.exe

      В качестве уточнения вышеперечисленного можно привести такой пример:
Пусть в локальной сети находятся два компьютера, на первом из которых установлен 
ключ от Project Expert 7 и менеджер лицензий, а на втором - ключ от Audit Expert 3 и 
менеджер лицензий (обе программы сетевые). 
Для первого компьютера необходимо настроить переменные так:
Для Project Expert:
HL_SEARCH=IP,IPX
HLS_IPADDR=<имя компьютера 1>
Ярлык запуска программы настраивается как «…run.exe» pew32.exe
Для Audit Expert:
HL_SEARCH=IP,IPX
HLS_IPADDR=<имя компьютера 2>
Ярлык запуска программы настраивается как «…run.exe» Audit32.exe
Для второго компьютера необходимо настроить переменные так:
Для Project Expert:
HL_SEARCH=IP,IPX
HLS_IPADDR=<имя компьютера 1>
Ярлык настраивается как «…run.exe» pew32.exe
Для Audit Expert:
HL_SEARCH=IP,IPX
HLS_IPADDR=<имя компьютера 2>
Ярлык настраивается как «…run.exe» Audit32.exe
В представленном примере, если одна из двух программ была бы локальной, то у нее 
переменной HL_SEARCH нужно бы было присвоить значение 378p,278p,3BCp.
